The Problem

Crisis decisions are made in silos.
Every expert sees one piece of the puzzle.

  • Health officials prioritize lockdowns — ignoring economic collapse
  • Economic advisors push to reopen — ignoring mortality spikes
  • Safety chiefs warn about riots — nobody listens until it's too late
  • No single perspective is correct — trade-offs are inevitable

Our Solution

A decision laboratory where
AI agents argue with real data

How It Works

The Decision Loop

1
Simulation runsSIR model spreads infection across 36 zones, 1 day per second
2
Every 5 days → Council convenesSituation report with zone-by-zone data sent to all 4 agents
3
Agents debate with real dataEach agent cites infection counts, hospital %, zone names — no generic advice
4
Coordinator issues decisionStructured: DECISION → REASONING CHAIN → CONCRETE ACTIONS → CONFIDENCE %
5
Actions execute on simulationLockdowns, vaccinations, hospital expansion modify live state instantly

Dynamic Economy & Morale

Economy drops from lockdowns + infections + hospital overflow. Morale drops from deaths + lockdown fatigue + fear. Both can go negative (-50).

Architecture

End-to-End Data Flow

User Input (Play / Advisory / Crisis)
  ↓
App.jsx — State Manager + Tick Loop
  ↓
simulation.js — SIR Model + Economy/Morale
  ↓ (every 5 days)
agents.js — Build Situation Report
  ↓
Gemini 2.5 Pro API — Streaming Response
  ↓
parseDecisionAction() — Extract Actions
  ↓
applyDecision() — Modify Simulation State
  ↓
UI Updates — Grid + Charts + Gauges

Situation Report

500-word brief sent to each agent: total infections, zone-by-zone hotspots, hospital capacity, economy/morale index, previous decisions and outcomes.

Agent Chaining

Health sees data → Economy sees data + Health's argument → Safety sees all → Coordinator weighs everything + user advisory.

Smart Fallback

If API rate-limited: intelligent mock responses using REAL simulation data. References actual zone names, numbers, advisory text. Indistinguishable from live AI.
